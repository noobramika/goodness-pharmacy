# Goodness Pharmacy Management System

## Overview

This repository contains the source code for the Goodness Pharmacy Management System, a desktop application developed by a Group of Students under - Object Oriented Programming with C# module. The software provides functionalities for inventory management, sales, purchases, customer and supplier management, and user administration.

## System Scope

The Goodness Pharmacy Management System is designed to streamline the inventory and sales management process for Goodness Pharmacy, a small-scale pharmacy and grocery store. The system offers features for inventory management, sales management, purchase management, customer management, supplier management, and user management.

## Technology Used

The software was developed using the following technologies:

- C# (.NET)
- Microsoft Visual Studio IDE
- Bunifu UI Framework (Front-End)
- Figma
- MySQL Database (Back-End)
- GitHub (Version Control and Collaboration)

## How the System Works

The Goodness Pharmacy Management System allows employees of Goodness Pharmacy to perform various tasks related to the day-to-day operations of the pharmacy. It provides a user-friendly interface for managing inventory, handling sales, processing purchases, managing customers and suppliers, and administering user roles.

### Inventory Management

Employees can add, update, and delete (CRUD OPERATIONS) items from the inventory. The system tracks crucial information such as medicine name, ID, expiry date, and quantity. It also alerts users when the stock of a particular item is running low.

### Sales Management

Users can create and process sales transactions, add multiple items to a single sale, apply discounts, and generate printable receipts. The system automatically updates the inventory after each sale.

### Purchase Management

Employees can manage purchase orders, select suppliers, and input relevant purchase information.

### Customer and Supplier Management

The system allows for the registration and management of customer and supplier information, including details such as names, addresses, and contact numbers.

### User Management

The system supports two user roles: Administrator and Employee. Administrators have access to additional features for managing employees and system administrators. Employees can perform standard operations within their defined permissions.
